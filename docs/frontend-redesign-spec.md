## 3. 功能規格

### 3.1 維護排程（Q2–Q5、Q14b）

- **本質：純系統建議、唯讀**。甘特圖上每一格都是 ROI 引擎算出的「清潔建議窗口」；人工確認/編輯流程列 N（願景）。
- **建議動作空間 = {PP, UWC, UWC+PP}**（M）：依船殼/螺旋槳歸因份額與各動作成本常數選最佳動作。效果模型：PP 抹除螺槳份額、UWC 抹除船殼份額、UWC+PP 兩者皆除。需擴充 ROI 引擎（三組成本參數＋歸因份額輸入）。
- **DD 唯讀**：資料中已知的進塢畫在甘特圖上當既定事件，不由系統建議。**UWI 附註**：不確定性高時顯示「建議先檢查」文字。
- **效果指標層級（Q4，命題對齊）**：主 KPI = **Speed Loss 預期回復（pp）**；副指標 = 回本天數、每月節省 USD、每日節省噸油。三者同源（同一計算鏈的不同單位）。
- **形式（Q5）**：總覽的預設收合甘特圖為全船隊每船一列並保留當次排序；決策頁只顯示總覽選定船舶的一列，不顯示排序。兩頁固定 100% 時間尺度，不提供縮放或前／中／後捲動控制。橫軸未來 180 天、按動作類型分色、今日垂直線、DD 固定塊；點色塊開建議詳情。**附資料表格 fallback**（a11y 硬需求）。
- **遞補建議（Q14b，M 輕量版）**：每個建議窗口點開顯示靜態「最佳遞補船」標註（同期每日超額成本次高者）。互動式拖拉重排／真實檔期容量列 N（資料中無廠商產能，不造假）。

### 3.2 單船儀表板（Q9/Q9b、Q14b）

- 點進一艘船＝該船的儀表板，**不是表格堆**：
  - **頂部 KPI 儀表磁磚**：最新正午日報＋衍生指標——今日均速、今日油耗（DailyFOC）、當日 SL%、超額油耗、風級、距上次清潔天數＋最近清潔動作。每磁磚附 30 天 sparkline。
  - **磁磚全做＋使用者可自訂顯示哪些**（顯示/隱藏設定存 localStorage）。
  - **中段主圖**：SL 趨勢＋水下事件標註＋清洗門檻線＋多模型預測曲線（見 3.3）。
  - **延遲代價卡（M）**：最佳清洗時機（可能已過）、「若當時清洗至今已省 $Y／已多花 $Y」、「現在洗 vs 再拖 30 天」對比。與既定待辦「反事實清潔效益」同源（fleet.csv 已有數字）。
  - **下一步建議卡**：動作＋窗口＋SL 回復主 KPI＋經濟副指標（與排程頁同一資料源）。
  - **可展開 30 天日誌表格**：正午日報逐列（日期/均速/DailyFOC/風級/全速時數/當日 SL/超額油耗），水下事件以強調列插入同一時間序。a11y 與細查用。
- **日報上傳聯動（既定待辦）**：`POST /api/noon-report` 一筆進來 → KPI 磁磚、主圖、日誌表即時更新（aria-live 播報），demo 亮點。

### 3.3 多模型預測（Q10、Q11）

- **模型註冊表** `GET /api/models`：id、名稱、描述、指標（如驗證 MAPE）、是否需要船速輸入、預設主模型標記。比賽版至少 2 個模型（現有線性外推 baseline ＋ 隊友新訓模型），介面支援 N 個。
- **主模型（primary）驅動下游**：排程建議、ROI、KPI、延遲代價全部跟「當前主模型」走，UI 標註依據模型。切換主模型 → 全站重算（demo 亮點：「換模型，建議清洗日從 8/10 提前到 7/28」）。
- **比對模型僅視覺**：可勾選多個疊加顯示；比對曲線只畫中線不畫誤差帶（誤差帶只給主模型）。
- **情境船速（Q10 = A 案）**：未來預測的假設營運船速，預設該船近 30 天均速，雙控件調整；只影響需要船速輸入的模型；**歷史 SL 計算（ISO 19030 基準）不動**。

### 3.4 燃油市場（Q12、Q13、Q13b）

- **資料源全套（M）**：
  1. **主**：Ship & Bunker 免費層爬取（8 大港 VLSFO/MGO/IFO380 日價＋近 11 日歷史），後端每 6–12 小時抓一次進快取；UI 顯著標註「Source: Ship & Bunker」＋連結（其 ToS 要求；demo 非商業使用＋低頻快取＋標註）。
  2. **趨勢**：USDA AgTransport 官方 API（全球 20 港平均，日頻、落後數週）畫長期歷史趨勢，零 key 零風險。
  3. **保底**：Yahoo Finance 期貨代理（`BZ=F`、`HO=F` 換算，以 S&B 快照校準係數），斷網/爬取失敗時用，標註 estimate。
  4. 高雄港：新加坡價＋固定溢價估算，標註 assumption。
  - 詳細調查與端點：`code/docs/fuel-price-data-sources.md`。
- **五油種呈現（M）**：HSHFO、VLSFO、ULSFO、LSMGO、BIO_HSFO 價格卡。對應：VLSFO↔S&B VLSFO、HSHFO↔S&B IFO380、LSMGO↔S&B MGO；**ULSFO 與 BIO_HSFO 為估算價，明確標註 estimate**。
- **價格量綱**：所有金額都明確顯示 `US$`，並依語意補上 `／日`、`／月`、`／30 天` 或 `／mt`；圖表 tooltip、座標軸與 a11y fallback 表格同樣適用。
- **船別加權有效油價驅動經濟計算（M）**：全站經濟數字（超額成本/ROI/排程/延遲代價）棄用 `VLSFO_PRICE_USD=600` 常數，改吃「該船近期日報五油種油耗佔比（熱值折算）加權的即時價」；UI 標註價格來源與時間戳。
- **手動覆寫（M）**：油價可雙控件覆寫做情境分析（「油價漲到 900 時提前清洗更划算」）。

### 3.5 經濟決策 What-if

- 加延遲代價視角（同 3.2 卡片的資料源）。

## 4. 設計系統規範

- **雙控件規範（Q8）**：所有數值參數一律「拉杆＋數字輸入框、雙向同步」共用元件（shadcn Slider + Input）；鍵盤上下鍵微調、超界 clamp。適用：SL 篩選、What-if 清洗日、情境船速、油價覆寫。
- **a11y = WCAG 2.1 AA（Q15，硬約束）**：
  - 文字對比 ≥ 4.5:1，深淺兩主題都要過。
  - 全站鍵盤可操作、focus 可見。
  - 所有圖表附資料表替代（甘特圖、趨勢圖、sparkline 群）。
  - 即時更新 aria-live；尊重 `prefers-reduced-motion`。
  - 顏色不當唯一訊息載體（狀態同時用圖示/文字；多模型曲線分色＋線型/標記區分）。
- 視覺方向：由設計版本提案（見 §6）。

## 6. 設計方向（已定案 2026-07-15）

三個 hi-fi 變體（`code/design-variants/`）比稿後選定 **B・艦橋儀控（Bridge Ops）**：

- **IA**：頂部三段故事流「① 總覽 Fleet → ② 診斷 Diagnose → ③ 決策 Decide」採內容寬度的步驟選單，不強制填滿頁首；AI 顧問與設定放在右上工具區，桌面／平板開啟顧問後頁首仍維持單行。總覽顯示預設收合的全船隊甘特圖；資料載入後先選取總覽第一列船舶並解鎖診斷／決策，兩頁明確顯示目前船名與 ID。**首頁頂部五油種 ticker**（含來源標註與暫停鈕）；警報使用預設收合的頂部通知匣（鈴鐺＋未讀數，僅由使用者開啟；點警報跳轉對應船）。原第 7 項水下影像判讀已依 2026-07-15 後續決策移除。
- **視覺 tokens**：儀器白底 `#F4F6F8`、主色深青 `#0E5E6F`、信號琥珀 `#C77400`、危險紅 `#A33B2E`；KPI＝等寬大讀數（tabular-nums）＋單位小標＋狀態形狀燈（▲●○■＋文字，不只靠色）；2px 實邊分區、無陰影堆疊；黑體字系、標題粗壯；圖表筆畫較粗、門檻以淡紅帶狀底呈現。
- **參考原型**：`code/design-variants/variant-b-bridge-ops.html`（單檔可開，含深色主題與全部關鍵畫面），實作以其為視覺基準翻成 React 元件。

品牌於 2026-07-15 後續決策更新為 **Oi! Hullwatch**，標語 **Oi! Save the Oil.**；Header 使用使用者提供的 Oi! 圖像資產，標語以灰色小字呈現。

其餘開放項（N 級清單）：人工確認排程流程、互動式遞補重排、DD 時機建議、單位切換器。
